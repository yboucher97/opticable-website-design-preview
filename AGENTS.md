# Repository Instructions

## Asset model

This repository now uses a strict separation between production assets, concept assets, and archived material.

- `assets/`
  Production runtime brand assets that are copied directly into the live build.

- `Images/production/`
  Production-only raw image sources used by `sitegen.py` for the live site.
  Only approved winners belong here.

- `Images/concepts/`
  Concept-only image work.
  Each concept has its own isolated image folders.

- `Images/archive/`
  Legacy, backup, and reference material that is not part of the live build.

- `dist/assets/`
  Generated output only. Never place source images here manually.

## Concept folders

Use the concept folder that matches the active branch:

- `codex/concept-01` -> `Images/concepts/concept-01/` and `concepts/concept-01/`
- `codex/concept-02` -> `Images/concepts/concept-02/` and `concepts/concept-02/`

Inside each concept image folder:

- `generated-candidates/`
  Default destination for images generated by Codex.

- `manual-upload/`
  Manually added files for that concept.

- `selected/`
  Candidate images currently chosen for that concept.

Inside each text concept folder:

- `brief.md`
  The current concept direction, prompt, scope, and goals.

- `feedback.md`
  What the user liked, disliked, or wants changed.

## Default behavior

If the user asks for a new concept and does not specify a concept number, default to `concept-01`.

If the user asks for generated candidate images, place them in that concept's `generated-candidates/` folder.

If the user provides files manually for a concept, place them in that concept's `manual-upload/` folder.

If the user approves an image for active concept use, move it into that concept's `selected/` folder.

## Promotion rules

Never point `sitegen.py` at `Images/concepts/` or `Images/archive/`.

When a concept asset becomes a winner for production:

1. Copy or move it into `Images/production/`.
2. Update `sitegen.py` to reference the new production path.
3. Rebuild the site.

Production folders should contain winners only.

## Deprecated folders

`Images/future-use/` is deprecated.

Do not use it for new work unless a task explicitly requires backward compatibility.
